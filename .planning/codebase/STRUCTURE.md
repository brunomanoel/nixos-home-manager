## Key Locations

| What | Where |
|------|-------|
| Flake entrypoint | `flake.nix` |
| Add a new host (NixOS) | `hosts/<hostname>/default.nix` + entry in `flake.nix` |
| Add a new host (macOS) | `hosts/<hostname>/default.nix` + `darwinConfigurations` in `flake.nix` |
| Home config for a host | `home/bruno/<hostname>.nix` |
| Always-on home config | `home/bruno/global/default.nix` |
| New opt-in feature | `home/bruno/features/<domain>.nix` → import in per-host `.nix` |
| System-wide option (all hosts) | `hosts/common/global/default.nix` (Linux) or `darwin.nix` (macOS) |
| Opt-in system feature | `hosts/common/optional/<feature>.nix` → import in host `default.nix` |
| SSH / keychain config | `home/bruno/features/cli/ssh.nix` |
| Nix GC / settings (Linux) | `hosts/common/global/nix.nix` |
| Nix GC / settings (macOS) | `hosts/common/global/nix-darwin.nix` |

## Naming Conventions

- `default.nix` — aggregator or primary entry point for a directory
- `<hostname>.nix` — per-host home-manager composition file
- `darwin.nix` — macOS-specific sibling to a Linux `default.nix`
- `nix-darwin.nix` — macOS-specific nix settings (mirrors `nix.nix`)
- Feature files named after the tool/domain they configure: `zsh.nix`, `tmux.nix`, `git.nix`

## Adding New Hosts

**NixOS:**
1. Create `hosts/<hostname>/default.nix` importing `../common/global` and `../common/users/bruno`
2. Create `home/bruno/<hostname>.nix` with desired feature imports
3. Add to `flake.nix` under `nixosConfigurations`

**macOS (nix-darwin):**
1. Create `hosts/<hostname>/default.nix` importing `../common/global/darwin.nix` and `../common/users/bruno/darwin.nix`
2. Create `home/bruno/<hostname>.nix` with desired feature imports
3. Add to `flake.nix` under `darwinConfigurations` with `system = "aarch64-darwin"` (Apple Silicon) or `"x86_64-darwin"` (Intel)
