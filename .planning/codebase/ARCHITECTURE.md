# Architecture

**Analysis Date:** 2026-03-06

## Pattern Overview

**Overall:** Nix Flake-based multi-host dotfiles monorepo

**Key Characteristics:**
- Single `flake.nix` is the root composition point, declaring all system and home-manager outputs
- Configuration is split into two orthogonal trees: `hosts/` (system-level, NixOS/nix-darwin) and `home/` (user-level, home-manager)
- Cross-cutting concerns are addressed by a `common/` layer shared across hosts
- Features are opt-in modules composed via `imports` lists in per-host entry point files
- Platform differences (Linux, macOS, WSL) are handled with `lib.mkIf pkgs.stdenv.isLinux` / `isDarwin` guards inside shared modules, or via platform-specific sibling files (e.g., `darwin.nix` next to `default.nix`)

## Layers

**Flake Root:**
- Purpose: Declares all inputs (nixpkgs, home-manager, nix-darwin, etc.) and outputs (nixosConfigurations, darwinConfigurations, homeConfigurations, devShells)
- Location: `flake.nix`
- Contains: Input pins, output combinators, per-host instantiation calls
- Depends on: All other layers (referenced by path)
- Used by: `nixos-rebuild`, `darwin-rebuild`, `home-manager` CLI commands

**Host System Layer:**
- Purpose: Machine-specific NixOS or nix-darwin system configuration — hardware, bootloader, networking, system services
- Location: `hosts/<hostname>/default.nix`
- Contains: Hardware imports, boot config, host-specific services, `system.stateVersion`
- Depends on: `hosts/common/` modules for shared settings; hardware-configuration files generated by NixOS
- Used by: `flake.nix` nixosConfigurations / darwinConfigurations entries

**Common System Layer:**
- Purpose: System-level config shared across all hosts
- Location: `hosts/common/`
- Contains:
  - `global/` — applied unconditionally: nix settings, locale, fonts, zsh system shell, CUDA binary cache, nix-darwin equivalents
  - `optional/` — opt-in features: GNOME, Docker, VirtualBox, gaming
  - `users/bruno/` — user account declaration and home-manager wiring
- Depends on: flake inputs (home-manager module, hardware modules)
- Used by: host `default.nix` files via `imports`

**User Home Layer:**
- Purpose: Home-manager configuration; user-space packages and program settings
- Location: `home/bruno/`
- Contains:
  - Per-host entry points (`predabook.nix`, `wsl.nix`, `mac.nix`) — import lists that compose features for that machine
  - `global/` — always-on home config: CLI feature set, fonts, nix settings, session variables
  - `features/` — opt-in home modules organized by domain
- Depends on: nixpkgs (for packages), home-manager module system
- Used by: `hosts/common/users/bruno/default.nix` via `home-manager.users.bruno = import .../home/bruno/<hostname>.nix` and standalone `homeConfigurations` in `flake.nix`

**Feature Modules:**
- Purpose: Discrete, self-contained configuration units that can be selectively imported
- Location: `home/bruno/features/`
- Contains: Domain-grouped `.nix` files (cli, dev, games, desktop features)
- Depends on: pkgs, config, lib passed via module system
- Used by: per-host entry points and `home/bruno/global/`

## Data Flow

**System Rebuild (NixOS):**

1. Operator runs `nixos-rebuild switch --flake .#predabook` (or via `nh os switch`)
2. Nix evaluates `flake.nix` → `nixosConfigurations.predabook` → `hosts/predabook/default.nix`
3. Host imports `hosts/common/global` (nix settings, locale, fonts, zsh, cuda) and `hosts/common/users/bruno`
4. `hosts/common/users/bruno/default.nix` declares the system user and wires `home-manager.users.bruno` to `home/bruno/predabook.nix`
5. `home/bruno/predabook.nix` composes `global/` + selected `features/` modules
6. Nix evaluates the full module tree, merges attribute sets, and builds the system closure

**macOS Rebuild:**

1. Operator runs `darwin-rebuild switch --flake .#mac`
2. `flake.nix` → `darwinConfigurations.mac` → `hosts/mac/default.nix`
3. Imports `hosts/common/global/darwin.nix` (darwin-specific nix/font/zsh/pam settings) and `hosts/common/users/bruno/darwin.nix`
4. Same home-manager wiring pattern as Linux, using `home/bruno/mac.nix` as feature composition entry point

**Standalone Home-Manager:**

1. Operator runs `home-manager switch --flake .#bruno@wsl`
2. `flake.nix` → `homeConfigurations."bruno@wsl"` → `home/bruno/wsl.nix` directly (no system layer involvement)

**Module Composition:**
- All configuration flows through Nix module system `imports` lists — no runtime imperative logic
- `lib.mkIf pkgs.stdenv.isLinux` / `isDarwin` guards inside modules handle cross-platform attribute inclusion at evaluation time
- `lib.optionals` is used in package lists for platform-conditional package inclusion

## Key Abstractions

**Feature Module:**
- Purpose: A self-contained home-manager configuration unit for a logical domain
- Examples: `home/bruno/features/cli/default.nix`, `home/bruno/features/git.nix`, `home/bruno/features/games/default.nix`
- Pattern: A `.nix` file returning a module attrset with `imports`, `home.packages`, and `programs.*` keys; imported by the per-host entry point when that feature is desired on that machine

**Per-Host Entry Point (home):**
- Purpose: Selects which features apply to a specific machine
- Examples: `home/bruno/predabook.nix`, `home/bruno/wsl.nix`, `home/bruno/mac.nix`
- Pattern: Minimal file containing only an `imports` list; no direct configuration — all config lives in imported modules

**Optional System Module:**
- Purpose: A system-level feature enabled per-host via explicit import
- Examples: `hosts/common/optional/gnome.nix`, `hosts/common/optional/docker.nix`, `hosts/common/optional/gaming.nix`
- Pattern: A NixOS module file that is unconditionally applied if imported; hosts choose to import or omit it

**Global Baseline:**
- Purpose: Config guaranteed to apply on every host without explicit opt-in
- Examples: `home/bruno/global/default.nix` (always imported from every per-host home entry point), `hosts/common/global/default.nix` (always imported from every host system config)
- Pattern: `global/default.nix` is the aggregation point that imports all sub-modules in the global set

## Entry Points

**NixOS System (predabook):**
- Location: `hosts/predabook/default.nix`
- Triggers: `nixos-rebuild switch --flake .#predabook`
- Responsibilities: Hardware config, bootloader (GRUB+EFI), kernel modules, networking, audio (pipewire), Wayland, power management

**NixOS System (wsl):**
- Location: `hosts/wsl/default.nix`
- Triggers: `nixos-rebuild switch --flake .#wsl`
- Responsibilities: WSL integration, VS Code Server, nix-ld, open-webui (Ollama web portal), Docker

**macOS System (mac):**
- Location: `hosts/mac/default.nix`
- Triggers: `darwin-rebuild switch --flake .#mac`
- Responsibilities: Hostname, stateVersion, delegates all config to `darwin.nix` and `darwin.nix` user module

**Home (predabook):**
- Location: `home/bruno/predabook.nix`
- Triggers: Invoked automatically during system rebuild via home-manager NixOS module, or standalone via `home-manager switch --flake .#bruno@predabook`
- Responsibilities: Composes full desktop feature set (git, utils, dev, games, GNOME, OBS, video editing, Ollama)

**Home (wsl):**
- Location: `home/bruno/wsl.nix`
- Triggers: System rebuild or standalone home-manager
- Responsibilities: Minimal — git, reverse-engineering tools

**Home (mac):**
- Location: `home/bruno/mac.nix`
- Triggers: Darwin rebuild or standalone home-manager
- Responsibilities: Minimal — git only

## Error Handling

**Strategy:** Nix evaluation — type errors and missing attributes fail at build time, not runtime

**Patterns:**
- `lib.mkDefault` used to set overridable defaults (e.g., username, homeDirectory, stateVersion) that hosts can override
- `lib.mkIf` used to conditionally enable attributes without causing evaluation errors on unsupported platforms
- `ifTheyExist` helper in `hosts/common/users/bruno/default.nix` filters user groups to only those that exist on the current host, preventing errors on hosts without certain services
- Commented-out config (`# ./features/ollama.nix`) is the mechanism for disabling features without deleting them

## Cross-Cutting Concerns

**Nix Store / Garbage Collection:** Configured in `hosts/common/global/nix.nix` (Linux) and `hosts/common/global/nix-darwin.nix` (macOS) — automatic GC weekly, keep last 3 generations, auto-optimize store

**Package Allowances:** `nixpkgs.config.allowUnfree = true` set in both system (`hosts/common/global/`) and home (`home/bruno/global/default.nix`) layers

**Shell (system-level):** zsh enabled system-wide in `hosts/common/global/zsh.nix`; user zsh configuration lives in `home/bruno/features/cli/zsh.nix`

**Home-manager Integration:** Wired from the system layer via `inputs.home-manager.nixosModules.home-manager` (Linux) or `inputs.home-manager.darwinModules.home-manager` (macOS) in the respective global modules; `home-manager.useUserPackages = true` is consistently set
