== DECISÕES TOMADAS ==
- ARM Ampere A1 (4 cores, 24GB, DDR4 ~25GB/s) é LENTO DEMAIS para embedding inference com inputs reais.
  - 50 chars: 1.5s | 500 chars: 11s | 1000 chars: 23s | 2000 chars: 46s | 3000 chars: 71s
  - Escala ~23ms/char. Memory bandwidth é o bottleneck (~25 GB/s vs ~273 GB/s Apple M4 Pro).
  - Leitura (recall) com queries curtas (50-200 chars) funciona: 0.5-2.3s. Aceitável.
  - Escrita (indexação) com chunks de 3000 chars: 71s. Estoura timeout 60s do local-rag.
- local-rag: MAX_CHUNK_CHARS=3000 e DESC_CONCURRENCY=5 hardcoded. Patches locais funcionam (2000 chars, concurrency 1) mas são frágeis (npx reinstala).
- Cloudarm usa nixpkgs-stable (25.11) com overlay unstable para ollama, qdrant, chromium.
- Reestruturação dotfiles feita: desktop/server separados, HM per-host.
- ai-services.nix reabilitado no predabook (Ollama+Qdrant+Open WebUI local).
- MCPs no cloudarm precisam de bash+nodejs+coreutils no PATH dos systemd services.
- uvx no NixOS precisa de programs.nix-ld.enable = true.
